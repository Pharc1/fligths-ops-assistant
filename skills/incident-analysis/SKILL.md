# Incident Analysis

## Quand utiliser ce skill
Quand un pilote ou technicien soumet un rapport décrivant une anomalie observée en vol ou au sol.

## Raisonnement attendu

1. Identifier le système concerné à partir de la description (hydraulique, avionique, moteur, train d'atterrissage, etc.)
2. Chercher dans la documentation technique les procédures et limites liées à ce système
3. Croiser avec l'historique des incidents pour voir si ce type d'anomalie a déjà été rencontré sur la flotte
4. Formuler un diagnostic : cause probable + niveau de confiance
5. Proposer des actions correctives classées par priorité

## Comment présenter les résultats

Dès que tu cites une valeur chiffrée ou une procédure issue de la documentation, montre la source — ne jamais paraphraser sans référence visuelle.

Si tu trouves des incidents similaires dans l'historique, présente-les de façon structurée en mettant en évidence les entrées anormales.

Si l'incident mentionne des valeurs mesurées (pression, température, usure), affiche-les avec leurs limites réglementaires pour que le technicien voit immédiatement si c'est dans les normes.

## Règles

- Ne jamais conclure sans avoir cherché dans la documentation
- Si les résultats sont insuffisants, le dire explicitement plutôt qu'inventer
- Rester concis et professionnel — le technicien a besoin d'actions, pas d'un cours
- Si la situation semble nécessiter une escalade (sécurité vol), le mentionner en premier
- Terminer par un rapport structuré avec diagnostic, actions recommandées et niveau de confiance
